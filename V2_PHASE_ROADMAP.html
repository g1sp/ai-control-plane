🎯 Overview

After v1.1 security hardening, Phase 2 focuses on enterprise readiness: multi-tenant isolation, configuration hot-reload, webhook delivery, and schema migrations. These features enable production deployments at scale while maintaining backward compatibility.

🪝 Phase 2.3: Webhooks & Rate Limit Headers

Weeks 3-4 | Async Notifications + Cost Attribution

Async webhook delivery with retry logic. Standard rate limit HTTP headers. Detailed cost breakdown per request for accurate attribution.

☐
WebhookDelivery table — Track status, attempts, next retry, HMAC signature
☐
WebhookDeliveryService — Queue + retry with exponential backoff (1s, 4s, 16s)
☐
Webhook signing (HMAC-SHA256) — Sign payloads, clients verify with API key
☐
Rate limit response headers — X-RateLimit-Limit, -Used, -Remaining, -Reset
☐
HTTP 429 on limit exceeded — Include Retry-After header with seconds
☐
CostBreakdown table + attribution — Per-request model costs (input, output tokens × rate)
